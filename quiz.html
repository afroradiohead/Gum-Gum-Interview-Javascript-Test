  5. What is minification and why we should use it? Do you use it? How?
    • It compressses a bunch of characters to fit in a tight space without changing functionality
    • It increases loading times by decreasing file size
  6. What is the DOM? And the BOM? What's the difference? Give us a one-line example of using each one.
    • DOM: Document Object Model - Webpage Objects (html elements) i.e. document.getElementById("id")
    • BOM: Browser Object Model - Browser Objects (window, navigator, history, js objects) i.e. window.location.reload()
  7. Why is HTML5shiv important on some browsers?
    • Browsers that don't support html5 (internet explorer 8-) need this in order to allow devs to work with html5 properly
  8. How many ways to communicate between two different domain pages do you know? (client-side, of course)
    • If a page opened another page it has access to the window.opener
    • An ajax request to another domain if the domain's says it's okay in header
    • An ajax request to your own backend script that crawls another domain
  9. Is document.write good or bad? When its good? When its bad?
    • Bad: Can clear the document if it happens after the page loads
    • Good: Writing things into an iframe
  10. What do you understand for self-invoking function? Do you use them? Why?
    • It runs immediately after created. Best used in to calls timers as setInterval has a delay
    • I think I used it in a game once, but I know I've used it while making jquery plugins.
  11. Do you use JSLint or JSHint? What about strict mode? Why?
    • I don't use JSLint or JShint or strict mode. I haven't had the need to since I try to avoid bad practices, but I would consider using it while working with other devs.
  12. What do you know about memory leaks?
    • Google Chrome lets me snapshot the heap and find potential memory issues. I haven't worked on heavy enough web application to find it worthwhile to dive super deep yet.
  13. Have you heard of the "Repainting" problem on the browser? Tell us about it.
    • I've ran into repainting performance issues with box-shadows & box-radiuses.
    • Basically everytime the page looks different (from scrolling or dom manipulation) the browser repaints, which if seriously slow will mess up the ux.

    function blackBox(){
        var argumentList = Array.prototype.slice.call(arguments);
        var argumentText = argumentList.join("-");
        argumentText = argumentList.length == 3 ? argumentText + "!" : argumentText;
